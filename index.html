Save Chat Transcripts in PHP application

As a developer supporting legacy products I fixed a chat application so the conversation transcripts would be saved to the database and emailed to the users.The problem was that the query used to save the chat trancripts to the database was wrong, it allowed the chat trancript to escape the query, preventing it from being saved to the database. Using PHP parameter binding, I encapsulated the data before running the SQL query. This improved security by preventing SQL injection and soved the problem now that the input data would not interfere with the query. The local development was done in a docker container and the code was deployed to a Linux server running a LAMP (Apache, MySQL, & PHP) stack.
